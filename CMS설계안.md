# LUV WORLD — CMS 전환 설계안

> 상태: **설계 채택 / 구현 보류** (2026-06-11 결정)
> 지금은 `admin.html` + `data.json` 구조로 운영하고, Netlify 첫 배포 후
> 공지·일정·링크를 몇 번 실제로 수정해본 뒤 **수정이 잦아지면 Sveltia CMS로 전환**한다.

---

## 1. 목표

코드를 열지 않고, **홈페이지 관리자 화면에서** 공지사항·방송일정·링크·최근영상을
수정하고 → **저장만 누르면 인터넷 사이트에 자동 반영**되게 한다. (+ 로그인 잠금)

---

## 2. 현재 구조가 CMS에 적합한가? → ✅ 매우 적합

| 현재 상태 | CMS 전환 관점 |
|----------|--------------|
| 데이터가 `data.json` 한 파일에 모임 | CMS가 편집할 대상 파일이 명확 |
| LINKS·NOTICES·SCHEDULE·CLIPS 등 9개 구조 표준화 | CMS 설정에 1:1 매핑만 하면 됨 |
| `script.js`가 data.json을 읽어 화면을 그림 | CMS가 data.json만 고치면 자동 갱신 (렌더 코드 수정 불필요) |
| 프레임워크 없는 정적 사이트 | Git 기반 CMS와 궁합 최상 |

→ 데이터 구조(LINKS·NOTICES·SCHEDULE·CLIPS 등)는 **전환 시에도 그대로 유지**된다.

---

## 3. 작동 개념

```
[지금]  admin.html 수정 → data.json 다운로드 → 폴더에 넣기 → 다시 업로드   (수동 3단계)

[CMS]   관리자 화면 수정 → 저장 클릭 → 끝
        (CMS가 data.json을 GitHub에 자동 저장 → Netlify 자동 재배포 → 사이트 자동 반영)
```

핵심 차이: **저장만 누르면 인터넷까지 자동**, 그리고 **로그인 잠금**이 생긴다.

---

## 4. 전환 작업 목록

| 단계 | 작업 | 담당 |
|------|------|------|
| 1 | GitHub 저장소 생성 + luv-website 업로드 | 주인님(안내) |
| 2 | Netlify를 드래그&드롭 → GitHub 연동 배포로 전환 | 주인님(안내) |
| 3 | `admin/` 폴더에 CMS 파일 2개 추가 (`index.html` + `config.yml`) | Claude |
| 4 | `config.yml`에 data.json 9개 항목 매핑 | Claude |
| 5 | 로그인 인증 설정 (GitHub 로그인) | 함께 |
| 6 | 테스트 (수정→저장→자동반영) | 함께 |

코드 작업(3·4)은 Claude가, 주인님은 계정 만들기·연결 위주.

---

## 5. 난이도 · 장단점 비교

| 방식 | 난이도 | 자동반영 | 로그인 | 비용 | 비고 |
|------|:---:|:---:|:---:|:---:|------|
| 지금 admin.html 유지 | ⭐ 매우 쉬움 | ❌ | ❌(주소 숨김) | 0원 | 이미 작동 |
| **Sveltia CMS** (추천) | ⭐⭐ 보통 | ✅ | ✅ GitHub | 0원 | Decap 호환·로그인 쉬움·모바일 OK |
| Decap CMS | ⭐⭐⭐ 약간 복잡 | ✅ | ✅ | 0원 | 유명하나 로그인 셋업 까다로움 |
| 외부 CMS(Sanity 등) | ⭐⭐⭐⭐ | ✅ | ✅ | 무료~유료 | 우리 규모엔 과함 |

### ⚠️ 현실 체크 (전환 시 확인 필요)
- Decap의 기존 로그인 방식인 **Netlify Identity**가 신규 사용 제한 방향으로 바뀐 상태
  (전환 시점에 정책 재확인 필요) → 그래서 **GitHub 계정으로 바로 로그인되는 Sveltia CMS**가 더 안전·쉬움.
- 셋업에 **GitHub 계정**(무료) 필요 — 비개발자에게 유일한 진입장벽.
- 비용은 전부 **0원** (Sveltia·Decap·Netlify·GitHub 무료 범위, 별도 API 비용 없음).

## 6. 추천 — 단계적 접근

```
1) 지금         : admin.html로 내용 채우고 → Netlify 첫 배포 (수동)
2) 운영 익숙해지면 : 수정이 잦아질 때 → Sveltia CMS 전환 (자동반영)
```

**먼저 배포해 운영해보고, 불편해지면 그때 CMS** 가 가장 효율적.
CMS의 진짜 장점(자동반영·로그인)은 "인터넷에 올린 뒤 자주 수정할 때" 의미가 커진다.

### 전환을 시작할 신호 (이 중 하나라도 해당되면 그때 진행)
- 공지/일정을 일주일에 여러 번 고치게 된다
- 파일 다운로드→폴더 교체→재업로드가 번거롭게 느껴진다
- PC 말고 폰에서도 수정하고 싶다
- 다른 사람과 함께 관리하게 된다
